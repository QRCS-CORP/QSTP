\doxysubsection{Detailed Description}
QSTP logging functions. 

This header defines the internal logging functions for the Quantum Secure Tunneling Protocol (QSTP). These functions provide a basic logging subsystem to record operational events, errors, and diagnostic messages during QSTP execution. The logging system is designed for debugging and monitoring purposes and includes functionality to initialize the log, write messages, read and print the log, reset (clear) the log, and retrieve the current log file size.

The logging subsystem uses several defined constants\+:


\begin{DoxyItemize}
\item {\ttfamily QSTP\+\_\+\+LOGGING\+\_\+\+MESSAGE\+\_\+\+MAX\+:} Defines the maximum allowed length (in characters) for a single log message.
\item {\ttfamily QSTP\+\_\+\+LOGGER\+\_\+\+PATH\+:} The default directory path where the QSTP log file is stored.
\item {\ttfamily QSTP\+\_\+\+LOGGER\+\_\+\+FILE\+:} The default log filename.
\item {\ttfamily QSTP\+\_\+\+LOGGER\+\_\+\+HEAD\+:} The header string written to the log file at initialization, which typically includes version information.
\end{DoxyItemize}

The logger also includes a built-\/in manual test ({\ttfamily \doxylink{logger_8h_a307e614ef1d21fa09fe670330a968ce4}{qstp\+\_\+logger\+\_\+test()}}) that exercises all of the logging functions to ensure proper operation.

\begin{DoxyNote}{Note}
These functions and constants are internal and non-\/exportable. 
\end{DoxyNote}
